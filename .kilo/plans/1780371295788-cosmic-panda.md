# Plan para MVP: Detección de Violencia en Tweets usando Modelos de HuggingFace en Español

## Objetivo
Crear un Producto Mínimo Viable (MVP) para analizar el nivel de violencia expresado en tweets en español, utilizando modelos pre-entrenados de HuggingFace.

## Características Propuestas del MVP

### Funcionalidad Principal
- **Análisis de violencia en texto**: Recibir un tweet y devolver una puntuación de violencia (0-1) o categoría (bajo, medio, alto)
- **Detección de tipos de violencia**: Identificar si la violencia es física, verbal, psicológica, etc. (si el modelo lo permite)
- **Interfaz simple**: Aplicación web basada en Gradio para ingresar tweets y visualizar resultados
- **Explicación de resultados**: Mostrar qué aspectos del texto contribuyeron a la puntuación de violencia

### Aplicaciones Potenciales
1. **Monitoreo de redes sociales**: Detectar contenido violento en tiempo real para plataformas
2. **Moderación de contenido**: Ayudar a identificar tweets que violen políticas de plataforma
3. **Análisis de conflictos**: Estudiar patrones de violencia en discusiones públicas
4. **Bienestar digital**: Alertar a usuarios sobre exposición a contenido potencialmente dañino
5. **Investigación social**: Estudiar expresiones de violencia en contextos específicos

## Enfoque Técnico

### Modelo Base
Utilizar modelos de HuggingFace especializados en español para tareas relacionadas con violencia:
- **Opción 1**: Modelos de detección de discurso de odio/toxicidad (ej: `Hate-speech-CNERGE/bert-base-uncased-hatexplain` adaptado al español o modelos específicos en español)
- **Opción 2**: Modelos de análisis de emociones que detecten ira, miedo o repulsión (indicadores indirectos de violencia)
- **Opción 3**: Fine-tuning ligero de un modelo existente si se tiene pequeño dataset etiquetado

### Componentes del MVP
1. **Backend**: 
   - Pipeline de HuggingFace `transformers` para carga y inferencia del modelo
   - Función de preprocesamiento específica para tweets (manejo de hashtags, mentions, emojis)
   - Función de postprocesamiento para convertir salida del modelo en puntuación de violencia interpretable

2. **Frontend**:
   - Interfaz Gradio con:
     - Área de texto para ingresar tweet
     - Botón de análisis
     - Visualización de resultados (puntuación, categoría, explicación)
     - Ejemplos pre-cargados de tweets con diferentes niveles de violencia

3. **Integración**:
   - Uso de `pipeline("text-classification")` o similar con modelo apropiado
   - Manejo de errores y casos edge (texto vacío, muy largo, etc.)

### Recursos Disponibles en el Workspace
- Entorno con `transformers`, `gradio`, `torch` ya instalado (según requirements.txt)
- Experiencia previa con Gradio y modelos HF en los notebooks del laboratorio
- Directorio `mi-space-sentimientos` con plantilla de app Gradio para análisis de sentimientos

## Pasos para Implementación

1. **Investigación de modelos** (1-2 horas):
   - Buscar en HuggingFace modelos en español para detección de violencia, odio o toxicidad
   - Evaluar modelos de emociones (pysentimiento/robertuito-emotion-analysis) como proxy
   - Seleccionar 1-2 modelos candidatos para probar

2. **Prototipo básico** (2-3 horas):
   - Crear script de prueba con pipeline de HF
   - Probar con ejemplos de tweets violentos y no violentos
   - Ajustar umbrales de clasificación

3. **Desarrollo de la app Gradio** (3-4 horas):
   - Adaptar plantilla existente de `mi-space-sentimientos`
   - Implementar función de análisis de violencia
   - Diseñar interfaz intuitiva con feedback visual
   - Añadir ejemplos informativos

4. **Pruebas y refinamiento** (2 horas):
   - Probar con diversos ejemplos de tweets
   - Verificar rendimiento y tiempos de respuesta
   - Ajustar según resultados

5. **Documentación básica** (1 hora):
   - README con instrucciones de uso
   - Explicación del modelo utilizado y sus limitaciones
   - Sugerencias para mejoras futuras

## Riesgos y Consideraciones

- **Limitaciones del modelo**: Los modelos de violencia pueden tener sesgos o no capturar todas las formas de violencia
- **Contexto**: La violencia en texto depende mucho del contexto que puede perderse en análisis aislado
- **Idiomática española**: Variaciones regionales en expresiones de violencia
- **Falsos positivos/negativos**: Necesidad de calibrar umbrales apropiadamente
